# CLAUDE.md

## 프로젝트: 데일리 카드 (Daily Card)

매일 하나의 질문에 답하고, 그 답을 카드로 만들어 저장/공유하는 서비스의 **프론트엔드**.

---

**브라우저에서 동작하는 웹앱(Web App)으로 만든다.** (네이티브 앱이 아님)

- 설치형 앱(PWA)이 아니라 **일반 웹앱**이다. `manifest.json`/Service Worker 등 설치·오프라인 기능은 만들지 않는다.
- 단, **UI는 모바일 앱처럼 보이도록** 디자인한다: 모바일 우선(반응형), 풀스크린 카드형 레이아웃, 화면 전환은 위저드 흐름에 맞는 부드러운 전환. (브라우저 탭 안에서 열리지만 "앱 같은 느낌"을 목표로 한다)

---

## 기술 스택

- **React 19** + **TypeScript** + **Vite 8**
- 패키지 매니저: npm (`package-lock.json` 기준)
- 린트: ESLint (flat config, `eslint.config.js`)

### 명령어

```bash
npm run dev      # 개발 서버 (vite)
npm run build    # 타입체크(tsc -b) 후 프로덕션 빌드
npm run lint     # eslint
npm run preview  # 빌드 결과 미리보기
```

---

## 화면 구성 (5개 + 부가 2개)

| # | 화면 | 내용 |
|---|---|---|
| 1 | 오늘의 질문 | 오늘의 질문 텍스트 + "답변하기" 버튼 |
| 2 | 답변 작성 | 질문 재노출 + 텍스트 입력칸 |
| 3 | 배경 선택 | 컬러/그라디언트/이미지 탭 + 카드 미리보기 |
| 4 | 내 카드 결과 | 완성 카드 + 다운로드 / 타인 답변 보기 |
| 5 | 타인 답변 보기 | 같은 질문의 다른 답변 1개 노출 |

**부가 요소**
- **완료 모달**: "오늘 이미 답변함" 상태일 때 1번 화면 위에 덮임. "카드 다시보기"로만 4번 이동 가능.
- **새 질문 토스트**: 날짜가 바뀌었을 때 상단에 잠깐 뜨는 알림.

---

## 기본 흐름

```
1.오늘의 질문 → 2.답변 작성 → 3.배경 선택 → 4.내 카드 결과 → 5.타인 답변 보기
                                                              ↓
                                                  "처음으로" → 1로 복귀(재진입 분기 다시 탐)
```

단계별 핵심 동작:
1. **1번**: 오늘 날짜에 매칭된 질문 표시.
2. **2번**: 답 입력 → "제출하기" → 3번. (이 시점에는 **아직 저장 안 됨**)
3. **3번**: 배경 선택 시 미리보기 즉시 반영. **"카드 만들기"를 누르는 순간 답변+배경이 저장됨** → 4번.
4. **4번**: 완성 카드 표시. **카드 영역을 클라이언트에서 이미지로 캡쳐**해 저장(모바일 공유 시트/데스크톱 다운로드)하고 완료 토스트를 띄운다.
5. **5번**: "다른 사람 답변 보기" → 같은 질문의 답변 중 무작위 1개. "처음으로" → 1번(재진입 체크 수행).

---

## 재진입 분기 (1번 화면 진입 시 항상 체크)

1번 화면(앱 진입/새로고침/"처음으로")에 들어올 때마다 두 가지를 확인한다.

**체크 1 — 새 질문 도착:** "마지막으로 본 날짜"와 오늘을 비교.
- 다름 → "새로운 질문이 도착했어요" 토스트 표시 후 "마지막 본 날짜"를 오늘로 갱신.
- 같음 → 알림 없음.

**체크 2 — 오늘 답변 기록:** "오늘 날짜로 저장된 답변이 있는가?" (서버 조회 기준)
- 있음 → **완료 모달** 표시 → "카드 다시보기"로 4번 이동. (하루 1회만 답변 가능)
- 없음 → 질문 화면 정상 표시.

> **주의:** 완료 모달은 **UX 안내일 뿐 강제 수단이 아니다.** 새로고침·devtools·API 직접 호출로 우회 가능하므로, 1일 1회의 **실제 강제는 서버**(`(익명ID, 날짜)` 유니크 제약)가 담당한다. 클라이언트는 "서버에 오늘 기록이 있을 때 모달을 띄우는" 역할만 한다.

---

## 질문 선정

- 운영자가 등록한 질문 풀에서 **서버가 미리 랜덤으로 선정한 오늘의 질문을 DB에서 조회**.
- 선정은 **서버에서 일괄 수행**되며, 같은 날짜에는 **모든 사용자가 동일한 질문**을 받는다. 날짜가 바뀌면 새로 선정된 질문.
- 클라이언트는 선정 로직에 관여하지 않고, 서버가 내려준 오늘의 질문을 그대로 표시한다.

---

## 서버 연동 항목

| 항목 | 처리 |
|---|---|
| 답변 + 배경 저장 | 서버 DB, 익명 식별자 단위. **`(익명ID, 날짜)` 유니크 제약으로 1일 1회를 서버에서 강제** (중복 시 409) |
| 타인 답변 노출 | 같은 질문의 답변 중 무작위 1개 조회 |
| 욕설 필터 | **2번 "제출하기"에서 1차 검사**(빠른 피드백) + 3번 저장 시 서버 최종 검증(이중). 차단 시 어떤 단어/재작성 유도를 사용자에게 표시 |
| 질문 목록 | 운영자 등록 질문 DB, **서버 기준 날짜(KST)** 매칭 조회 |
| 카드 다운로드 | **클라이언트에서 카드 영역을 이미지로 캡쳐**해 저장(모바일 Web Share 공유 시트 / 데스크톱 `<a download>`). 서버 연동 없음 — 클라이언트 전용 |

---

## 개발 시 주의사항

- **저장 시점**: 답변은 2번 "제출하기"가 아니라 3번 **"카드 만들기"** 시점에 저장된다. 혼동 금지.
  - 그 결과 2번→3번 사이 이탈/새로고침 시 답변이 유실된다. 이는 **의도된 동작**이되, 복구가 필요하면 작성 중 답변을 localStorage에 **드래프트로 임시 보존**한다.
- **하루 1회 제약**: 완료 모달은 **UX 안내**이며 강제 수단이 아니다. 실제 강제는 **서버**가 한다(위 "서버 연동 항목" 참고).
- **날짜 기준**: 모든 분기(질문 선정/재진입 체크/1일 1회)는 "오늘 날짜"에 의존한다. 이 날짜는 **클라이언트 `new Date()`가 아니라 서버 기준 단일 타임존(KST 자정)**으로 산정한다. 클라이언트는 서버가 내려준 날짜 키를 신뢰한다. (기기 시계 조작·타임존 차이로 인한 불일치 방지)
- **익명 식별자 방식(쿠키 전용)**: 사용자 식별은 **백엔드가 발급하는 서명된 httpOnly 쿠키 `anon_id` 하나**로만 이뤄진다. 클라이언트는 익명ID를 만들거나 헤더로 보내지 않으며(쿠키가 httpOnly라 JS로 읽을 수도 없다), 서버 쿠키가 단일 진실이다. 따라서 모든 API 요청은 `fetch(..., { credentials: 'include' })`로 쿠키를 자동 전송/저장해야 한다(백엔드 CORS도 `credentials: true` + 오리진 화이트리스트).
- **익명 식별자의 한계**: 쿠키 삭제·다른 브라우저·시크릿 모드에서는 새 쿠키가 발급되어 1일 1회 제약이 우회된다. 이는 **보장이 아니라 best-effort**이며, 실제 강제는 서버의 `(익명ID, 날짜)` 유니크 제약이 담당한다.

## 구현 방향 (아키텍처 결정)

스펙 외 구현 방식은 아래 기준을 따른다. (확정 전이면 이 섹션에서 결정 후 진행)

- **화면 전환**: 5단계는 순차 위저드 흐름이므로 **단일 상태(스텝) 기반 전환**을 기본으로 한다. URL 라우터는 딥링크 의미가 없어 우선순위 낮음. 단, 새로고침 복구·뒤로가기 동작 정책은 명시적으로 정한다.
- **상태에 따른 UI 분기**: 같은 화면이 상태에 따라 일부만 달라지면(레이아웃·데이터·대부분 UI 공유), **새 화면을 만들지 말고 하나의 컴포넌트 + prop/상태 분기**로 처리한다. (예: 카드 결과 화면의 완료 안내·카운트다운은 `CardResult`의 `completed` bool로 켠다) 그 bool은 **단일 진실**(서버/저장 상태)에서 파생한다. 단, 차이가 크거나 책임이 다르면 컴포넌트를 분리하고, 불리언이 여러 개 쌓여 조합이 복잡해지면 **variant enum**으로 바꾼다.
- **서버 상태 관리**: 질문 조회/답변 저장/타인 답변 조회 등 서버 데이터는 **React Query(TanStack Query) 등 데이터 패칭 라이브러리**로 로딩/에러/캐싱을 일관 처리한다. 수동 fetch 산발 금지.
- **API 계약**: 각 서버 연동 항목은 엔드포인트 형태와 **에러/로딩/네트워크 실패·욕설 차단·409(중복) 시 UI 분기**를 함께 정의한다.
- **카드 다운로드**: **카드 영역을 클라이언트에서 이미지로 캡쳐**해 저장한다(서버 생성/링크 공유 방식 아님). `html2canvas`/`html-to-image` 등 사용. 저장은 모바일 Web Share 공유 시트 / 데스크톱 `<a download>`, 완료 시 하단 토스트 노출.
